웹 브라우저는 DNS를 통해 알아낸 IP 주소로 서버를 찾아갔습다. 그런데 서버에 도착하니 여러 개의 문이 있고, 각 문에는 번호가 적혀 있었습니다.


이 번호가 적힌 각각의 문을 포트(port)라고 합니다. 서버는 하나지만, 통신의 종류에 따라 들어갈 수 있는 문이 여러 개로 나눠져 있는 거죠. 그렇다면 웹 브라우저는 어느 포트로 들어가야 할까요? 바로 80번 포트입니다. 그 이유는 URL에서 80번 포트가 생략되어 있기 때문입니다. 예를 들어 http://naver.com:80으로 접속해도 문제없이 접속되는 것을 확인할 수 있습니다.


80번 포트가 생략된 이유는, HTTP라는 프로토콜의 기본 포트 번호가 80번이기 때문입니다. 즉, 포트 번호를 따로 지정하지 않으면 프로토콜의 기본 포트가 사용됩니다. 프로토콜은 인터넷에서 통신할 때 지켜야 하는 표준 규약입니다. HTTP 외에도 다양한 프로토콜이 있으며, 대표적인 종류와 기본 포트 번호는 다음과 같습니다.


이제 웹 브라우저는 서버의 80번 포트를 통해 요청을 전달하면 됩니다. 그런데 “포트”라는 이름은 어디서 나온 걸까요? 포트를 직역하면 배가 드나드는 항구를 의미합니다. 이를 이해하기 위해 비유를 들어보겠습니다.


예를 들어 제주도에 무역회사가 있다고 해봅시다. 이 회사는 주로 미국과 중국에서 상품을 수입합니다. 회사에는 규칙이 있습니다. 하나의 항구에는 한 국가의 상품만 들어올 수 있다는 것인데요. 그래서 중국에서 온 상품은 1번 항구로, 미국에서 온 상품은 2번 항구로 들어옵니다. 즉, 배에 담긴 상품은 각 항구를 통해 제주도로 들어오는 것입니다.


서버도 마찬가지입니다. 클라이언트의 요청은 포트를 통해 서버로 들어옵니다. 그리고 각 포트에는 한 종류의 요청만 들어와야 합니다. 웹 브라우저를 통해 들어오는 요청은 80번 포트로 들어오고, FTP 클라이언트를 통해 들어오는 요청은 21번 포트로 들어옵니다. 즉, 구글 크롬에서 보낸 요청은 80번 포트로, 파일질라(FileZilla)에서 보낸 FTP 요청은 21번 포트로 들어오는 것이죠.


이제 웹 브라우저는 80번 포트를 통과해 마침내 웹 서버와 만날 수 있게 되었습니다.


그런데 웹 브라우저와 웹 서버는 어떤 방식으로 대화할까요? 바로 HTTP를 통해 통신합니다. 이 HTTP는 URL 앞부분에 있는 프로토콜과 같은 것입니다.
